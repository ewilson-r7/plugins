Description

TeamDynamix is an IT Service Management (ITSM) and Project Portfolio Management platform. This plugin allows users to create, read, update, and search tickets within a TeamDynamix instance, enabling automated ticketing workflows.

Key Features

  • Create TeamDynamix tickets to initiate remediation workflows
  • Get ticket details to monitor ticket status
  • Update existing tickets with new information or status
  • Search tickets by various criteria

Requirements

  • TeamDynamix instance base URL (e.g., https://yourorg.teamdynamix.com)
  • For Admin authentication — BEID and Web Services Key, found in TDAdmin under the organization detail page (requires the Add BE Administrators permission)
  • For User authentication — Username and password for a TeamDynamix service account with a security role that has ticket create/edit permissions and access to the target ticketing application
  • Application ID for the target TeamDynamix ticketing application, found in TDAdmin under Applications

Setup

Vendor Side — TeamDynamix API Configuration

Option 1: Admin Authentication (BEID + Web Services Key)

Admin authentication provides broad access and is suitable for integrations that need full ticket management capabilities.

  1. Log in to your TeamDynamix instance as an administrator
  2. Navigate to TDAdmin (the admin console)
  3. Go to the Organization detail page (usually accessible from the main admin dashboard)
  4. Locate the BEID (Back End Identifier) — this is a GUID displayed on the organization page
  5. Locate or generate the Web Services Key on the same page
  6. Ensure the admin account associated with these credentials is set to Active
Note: You need the Add BE Administrators permission to view the BEID and Web Services Key on the organization detail page.
Security: The Web Services Key grants broad API access. Store it securely and rotate it periodically. Do not share it in plain text.

Option 2: User Authentication (Username + Password)

User authentication uses a service account with scoped permissions, providing least-privilege access for ticket operations.

  1. Log in to TDAdmin
  2. Navigate to Users & Roles > Users
  3. Click Create > Create Service Account
  4. Provide a username (e.g., svc_insightconnect) and a strong password
  5. Assign a security role with the required permissions (see Permissions section below)
  6. Navigate to Applications > [Your Ticketing App] > Users & Roles
  7. Grant the service account access to the target ticketing application

Finding Your Application ID

  1. In TDAdmin, navigate to Applications
  2. Select the target ticketing application
  3. Note the numeric ID in the URL or application details panel

InsightConnect Side — Connection Configuration

Configure the following fields when creating the TeamDynamix connection in InsightConnect:

FieldTypeRequiredDescription
base_url string required The base URL of your TeamDynamix instance (e.g., https://yourorg.teamdynamix.com). Do not include a trailing slash.
auth_type string (enum) required Authentication method. Admin uses BEID and Web Services Key; User uses username and password. Default: Admin
beid string optional Back End Identifier (GUID) found in TDAdmin on the organization detail page. Required when auth_type is Admin.
web_services_key credential_secret_key optional Web Services Key found in TDAdmin on the organization detail page. Required when auth_type is Admin.
credentials credential_username_password optional Username and password for a TeamDynamix service account. Required when auth_type is User.
app_id integer required Numeric Application ID for the target ticketing application, found in TDAdmin under Applications.
How authentication works: The plugin authenticates against the TeamDynamix Web API at /api/auth (Admin) or /api/auth/login (User) to obtain a bearer token, which is then used for all subsequent API calls during the workflow execution.

Network Requirements

  • The InsightConnect orchestrator must be able to reach your TeamDynamix instance over HTTPS (port 443)
  • Target hostname: yourorg.teamdynamix.com (replace with your actual instance hostname)
  • If using an on-premises orchestrator behind a firewall, ensure outbound HTTPS is allowed to the TeamDynamix domain

Permissions

TeamDynamix supports two authentication methods with different permission scopes. Choose the method that aligns with your security requirements and the principle of least privilege.

Admin Authentication (BEID + Web Services Key)

Admin authentication provides broad access to the TeamDynamix API. The BEID and Web Services Key act as a system-level credential with full administrative scope.

CapabilityAccess LevelActions Enabled
Ticket Management Full (Create, Read, Update, Search) Create Ticket, Get Ticket, Update Ticket, Search Tickets
Cross-Application Access All applications Can operate on tickets in any ticketing application
User Context System-level Actions performed as the system, not a specific user
Caution: Admin credentials provide unrestricted API access. Use this method only when broad access is required. Consider using User authentication for tighter permission control.

User Authentication (Service Account)

User authentication uses a dedicated service account with a security role that can be scoped to specific permissions and applications. This is the recommended approach for production deployments following least-privilege principles.

Minimum Required Permissions

Permission / ScopeAccess LevelRationale / Actions Enabled
All: View All Accts/Depts Read Required to resolve account/department IDs when creating tickets
Tickets: Create/Edit Tickets Create + Update Enables Create Ticket, Update Ticket actions
Application Access Granted per-app Service account must be granted access to the target ticketing application

Recommended Permission Profiles

ProfilePermissionsUse Case
Read-Only View All Accts/Depts, View Tickets (application access) Get Ticket, Search Tickets only — monitoring and reporting workflows
Ticket Management View All Accts/Depts, Create/Edit Tickets (application access) All actions — full remediation workflow automation
Best Practice: Create separate service accounts for different privilege levels. Use a read-only account for monitoring workflows and a ticket-management account for remediation workflows.

Create Ticket

Create a new ticket in TeamDynamix. Use this action to open remediation tickets as part of automated vulnerability management workflows.

Input

NameTypeRequiredDescription
title string required Short title/subject of the ticket
description string optional Full description of the ticket
type_id integer required Numeric ID of the ticket type in TeamDynamix
form_id integer optional Numeric ID of the ticket form in TeamDynamix
account_id integer required Numeric ID of the account or department
priority_id integer required Numeric ID of the priority level in TeamDynamix
status_id integer required Numeric ID of the initial ticket status in TeamDynamix
requestor_email string required Email address of the ticket requestor
responsible_group_id integer optional Numeric ID of the group responsible for the ticket
additional_fields object optional JSON object of additional fields to include in the ticket payload (e.g., custom attributes)

Output

Returns the newly created ticket's ID, direct URL, and a success indicator.

{
  "success": true,
  "ticket_id": 12345,
  "ticket_url": "https://yourorg.teamdynamix.com/TDClient/42/Requests/TicketDet?TicketID=12345"
}

Get Ticket

Retrieve a TeamDynamix ticket by its numeric ID. Returns the full ticket object along with convenience fields for title and status.

Input

NameTypeRequiredDescription
ticket_id integer required The numeric ID of the ticket to retrieve

Output

Returns the full ticket object and extracted fields for quick access.

{
  "ticket": {
    "ID": 12345,
    "Title": "Test Ticket",
    "StatusName": "New"
  },
  "title": "Test Ticket",
  "status": "New",
  "ticket_id": 12345
}

Update Ticket

Update an existing TeamDynamix ticket. Only the fields provided will be modified; omitted fields remain unchanged.

Input

NameTypeRequiredDescription
ticket_id integer required The numeric ID of the ticket to update
title string optional New title for the ticket
description string optional New description for the ticket
status_id integer optional New status ID for the ticket
priority_id integer optional New priority ID for the ticket
additional_fields object optional JSON object of additional fields to update

Output

Returns a boolean indicating whether the update was successful.

{
  "success": true
}

Search Tickets

Search for tickets in TeamDynamix by text content and/or status. Returns a list of matching ticket objects.

Input

NameTypeRequiredDescription
search_text string optional Text to search in ticket titles and descriptions
status_id integer optional Filter by status ID
max_results integer optional Maximum number of tickets to return (default: 25)

Output

Returns the list of matching tickets and a count.

{
  "tickets": [
    {
      "ID": 12345,
      "Title": "Test Ticket"
    }
  ],
  "count": 1
}

Custom Types

This plugin does not define any custom output types. All outputs use standard types (string, integer, boolean, object, []object).
