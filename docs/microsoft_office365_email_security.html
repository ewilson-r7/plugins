Microsoft Office365 Email Security

Rapid7 InsightConnect Plugin — v4.1.0

Office365MicrosoftEmailPhishingRemediation

Description

The Microsoft Office 365 email security plugin adds utilities to help administrators manage their Office 365 instances. This plugin will allow administrators to take remediation actions across their organization.

Key Features

  • Block senders by domain or email address
  • Search for and optionally delete email across an organization
  • Get email trace information
  • Manage entries in the Tenant Allow/Block List

Requirements

  • An administrative account with modern authentication enabled
  • PowerShell connectivity to the Office 365 cloud servers
  • Azure AD App Registration with Exchange Online PowerShell and Security & Compliance PowerShell access
  • A self-signed certificate for certificate-based authentication (OAuth2)

Setup

Azure App Registration (OAuth2 Modern Authentication)

Since version 4.0.0, this plugin uses Modern Authentication (OAuth2) exclusively. Basic Authentication for Exchange Online has been deprecated by Microsoft. You must register an application in Azure AD and configure certificate-based authentication.

Step 1: Register an Application in Azure AD

  1. Sign in to the Microsoft Entra admin center
  2. Navigate to Identity > Applications > App registrations
  3. Click New registration
  4. Enter a name (e.g., "InsightConnect Office365 Email Security")
  5. Set Supported account types to Accounts in this organizational directory only
  6. Click Register
  7. Note the Application (client) ID and Directory (tenant) ID

Step 2: Generate a Self-Signed Certificate

Exchange Online PowerShell with modern auth requires certificate-based authentication. Generate a self-signed certificate:

# Generate a self-signed certificate (PowerShell)
$cert = New-SelfSignedCertificate -Subject "CN=InsightConnect O365 Email Security" `
    -CertStoreLocation "Cert:\CurrentUser\My" `
    -KeyExportPolicy Exportable `
    -KeySpec Signature `
    -KeyLength 2048 `
    -NotAfter (Get-Date).AddYears(2)

# Export the public key (.cer) for uploading to Azure
Export-Certificate -Cert $cert -FilePath ".\InsightConnectO365.cer"

# Export the private key (.pfx) for the plugin connection
Export-PfxCertificate -Cert $cert -FilePath ".\InsightConnectO365.pfx" -Password (ConvertTo-SecureString -String "YourPassword" -Force -AsPlainText)
Security Warning: Store the private key (.pfx) securely. Do not share it or commit it to source control. The certificate grants access to Exchange Online and Security & Compliance PowerShell on behalf of your organization.

Step 3: Upload the Certificate to Azure AD

  1. In the Azure portal, navigate to your App Registration
  2. Go to Certificates & secrets > Certificates
  3. Click Upload certificate
  4. Upload the .cer file (public key only)
  5. Note the Thumbprint value displayed after upload

Step 4: Assign API Permissions

  1. In your App Registration, go to API permissions
  2. Click Add a permission > APIs my organization uses
  3. Search for and add Office 365 Exchange Online
  4. Select Application permissions
  5. Add Exchange.ManageAsApp
  6. Click Grant admin consent

Step 5: Assign Exchange Online Admin Roles

  1. In Microsoft Entra admin center, navigate to Roles and administrators
  2. Assign the Exchange Administrator role (or a more scoped role) to the service principal
  3. For Security & Compliance operations, also assign Compliance Administrator
Tip: The plugin uses the account credentials (username/password) with modern authentication to connect to both Exchange Online PowerShell and Security & Compliance PowerShell. The account must have the appropriate Exchange Online roles assigned.

Network Requirements

  • Outbound HTTPS (port 443) to outlook.office365.com
  • Outbound HTTPS (port 443) to ps.compliance.protection.outlook.com
  • Outbound HTTPS (port 443) to login.microsoftonline.com (Azure AD authentication)
  • PowerShell remoting uses WinRM over HTTPS — ensure no proxy or firewall blocks these endpoints

This plugin requires specific Exchange Online and Security & Compliance roles depending on which actions you intend to use. Follow the principle of least privilege and assign only the roles needed for your workflows.

Block Sender Transport Rule

Add a domain or email address to a blocking transport rule in Exchange Admin Center. In Office 365 cloud, transport rules are limited to 8k of data or roughly 8100 characters (approximately 400 email addresses). If a new email address would break that limit, existing emails must be manually deleted to make room.

Note: Transport rules have a per-rule limit of 8192 characters and an overall limit of 20480 bytes for all rules combined. Monitor rule size to avoid failures.

Input

NameTypeRequiredDescription
domain_or_email_to_block string Required Domain or email address to block (e.g., example.com)
transport_rule_name string Required Name of transport rule (default: InsightConnect Block List)

Output

NameTypeRequiredDescription
result string Required Result of the operation

Example Output

{
  "result": "Success"
}

Email Compliance Search

Create a compliance search for provided email using the Security & Compliance Center. Supports Keyword Query Language (KQL) for building complex search queries.

Input

NameTypeRequiredDescription
compliance_search_name string Required Name of compliance search
content_match_query string Required Text search string or KQL query (e.g., from:example AND subject:"Click Me!")
exchange_location string Optional Exchange location (mailbox, group mailbox, or "All"). Default: All
query_timeout integer Optional Query timeout in minutes. Default: 60

Output

NameTypeRequiredDescription
affected_usersintegerRequiredNumber of affected users
emails_foundintegerRequiredEmails found that matched the query
users[]stringRequiredEmail addresses of all affected users

Example Output

  "users": ["user1@example.com", "user2@example.com"]
}

Email Compliance Purge

Purge emails found by a previously-run compliance search. The compliance search must already exist before running this action.

Destructive Action: This permanently deletes emails matching the compliance search. Ensure the search results are correct before purging. Deleted emails cannot be recovered.

Input

NameTypeRequiredDescription
compliance_search_name string Required Name of the compliance search to purge results from
query_timeout integer Optional Query timeout in minutes. Default: 60

Output

NameTypeRequiredDescription
successbooleanRequiredWhether the purge completed successfully

Example Output

{
  "success": true
}

Email Compliance Search and Purge

Combined action that searches for emails matching a KQL query and optionally deletes them. Set delete_items to true to execute the purge.

Destructive Action: When delete_items is true, matched emails are permanently deleted. Test with delete_items: false first to review results before purging.

Input

NameTypeRequiredDescription
compliance_search_name string Required Name of compliance search
content_match_query string Required Text search string or KQL query
delete_items boolean Optional Execute the delete action only if true. Default: false
query_timeout integer Optional Query timeout in minutes. Default: 60

Output

NameTypeRequiredDescription
successbooleanRequiredWhether the operation completed successfully

Example Output

{
  "success": true
}

Create Tenant Allow/Block List Entry

Add a new entry to the Tenant Allow/Block List to block or allow senders, URLs, file hashes, or IPs. Entries can be configured with an expiration or set to never expire (block entries only).

Input

NameTypeRequiredDescription
list_type string Required Type of entry to create. Enum: Sender, Url, FileHash, IP
entries []string Required Values to add. For Sender: email or domain. For URL: URL value. For FileHash: SHA256 hashes. For IP: IPv6 or CIDR ranges
action_type string Required Whether to create an Allow or Block entry. Enum: Allow, Block
no_expiration boolean Optional If true, entry never expires (block entries only). Default: false
expiration_days integer Optional Days until expiration (ignored if no_expiration is true). Default: 30
notes string Optional Optional notes to associate with the entry

Output

NameTypeRequiredDescription

Remove Tenant Allow/Block List Entry

Remove entries from the Tenant Allow/Block List by specifying the list type and values to remove. Values must match existing entries exactly.

Note: Removed entries take effect immediately. Ensure you are removing the correct entries as this action cannot be undone — you would need to re-create the entry.

Input

NameTypeRequiredDescription
list_type string Required Type of entries to remove. Enum: Sender, Url, FileHash, IP
entries []string Required Values to remove (must match existing entry values exactly)

Output

NameTypeRequiredDescription
successbooleanRequiredWhether the entries were removed successfully

Example Output

{
  "success": true
}

Troubleshooting

  • Queries are expected to be wrapped in quotes, e.g., subject:"a subject"
  • If the 8192 character (per rule) limit is reached when adding additional conditions/emails to an existing rule, conditions/emails within that rule will need to be deleted to allow for the addition of new conditions/emails.
  • If the 20480 bytes (overall limit for all rules combined) is reached when making a new rule, existing rules can be deleted to make space, otherwise deleting conditions/emails within existing rules will allow for the addition of a new rule.
