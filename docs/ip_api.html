Description

Geolocate IPv4 addresses, IPv6 addresses, and domain names using the ip-api.com service. This plugin leverages the ip-api.com REST API to return detailed geolocation and network metadata for any public IP or domain, including country, city, ISP, AS information, and proxy/VPN detection flags.

Key Features

  • Geolocate a single IP address or domain name
  • Geolocate up to 100 IPs or domains in a single batch request
  • Detect proxy, VPN, Tor exit nodes, and hosting/data-center IPs
  • Retrieve AS number, ISP, and organization information
  • Localized city and country names in 8 languages

Requirements

  • No API key required for the free tier
  • Rate limit of 45 requests/minute for single lookups (free tier)
  • Rate limit of 15 requests/minute for batch lookups (free tier)
  • Free tier requests must use HTTP (not HTTPS) — the API enforces this
  • Pro tier requires a purchased API key and supports HTTPS

Setup

Vendor Side Setup (ip-api.com)

The ip-api.com service offers two tiers:

Free Tier

  • No registration or API key required
  • Requests are made over HTTP only (HTTPS is not supported on the free tier)
  • Rate limited to 45 requests/minute for single lookups and 15 requests/minute for batch lookups
  • Exceeding the rate limit returns a 429 status and the IP is temporarily banned
  • Intended for non-commercial use
Tip: The free tier is sufficient for low-volume enrichment workflows (e.g., on-demand IP lookups from alerts). For production-scale automation, consider the Pro tier.

Pro Tier

  • Navigate to https://members.ip-api.com/signup to create an account
  • After registration, your API key is displayed on the dashboard
  • Pro requests use HTTPS: https://pro.ip-api.com/json/{query}?key=YOUR_KEY
  • Higher rate limits and no usage restrictions
  • Supports additional response fields (mobile, proxy, hosting detection)
Security: If using the Pro tier, store your API key securely. Never commit it to source control or share it in plaintext.

InsightConnect Side Setup

This plugin does not require a connection configuration. It calls the ip-api.com free-tier endpoint directly with no authentication.

Note: Because no connection is needed, you can use this plugin immediately after installation — no credential setup required.

Permissions

The ip-api.com service does not use role-based access control or API scopes. Access is governed by tier:

TierAuthenticationRate LimitCapabilities
Free None (IP-based tracking) 45 req/min (single), 15 req/min (batch) Full geolocation fields, proxy/VPN detection, AS info
Pro API Key (query parameter) Unlimited (fair use) All free tier fields + HTTPS + higher limits + commercial use
Least-Privilege Guidance: For non-production enrichment and testing, the free tier with no API key is appropriate. For production workflows processing alert volumes, use a dedicated Pro API key so you are not subject to the shared free-tier rate limit.
Note: The free tier tracks usage by source IP. If multiple integrations share the same egress IP, they share the rate limit. Consider the Pro tier if this is a concern.

Geolocate IP

Look up geolocation data for a single IPv4 address, IPv6 address, or domain name.

Input

NameTypeRequiredDescription
query string Required IPv4 address, IPv6 address, or domain name to look up (e.g. 8.8.8.8)
lang string Optional Language for localized city and country names. Default: en. Options: en, de, es, pt-BR, fr, ja, zh-CN, ru

Output

Returns a single geolocation_result object with location, network, and proxy-detection fields.

Example Output

{
  "result": {
    "status": "success",
    "country": "United States",
    "countryCode": "US",
    "region": "VA",
    "regionName": "Virginia",
    "city": "Ashburn",
    "zip": "20149",
    "lat": 39.03,
    "lon": -77.5,
    "timezone": "America/New_York",
    "isp": "Google LLC",
    "org": "Google Public DNS",
    "as_number": "AS15169 Google LLC",
    "query": "8.8.8.8",
    "mobile": false,
    "proxy": false,
    "hosting": true
  }
}

Geolocate Bulk

Look up geolocation data for up to 100 IPv4 addresses, IPv6 addresses, or domain names in a single request.

Input

NameTypeRequiredDescription
queries []string Required List of IPv4 addresses, IPv6 addresses, or domain names to look up (maximum 100)
lang string Optional Language for localized city and country names. Default: en. Options: en, de, es, pt-BR, fr, ja, zh-CN, ru

Output

Returns a list of geolocation_result objects, one per queried IP/domain.

Example Output

{
  "results": [
    {
      "status": "success",
      "country": "United States",
      "city": "Ashburn",
      "isp": "Google LLC",
      "query": "8.8.8.8",
      "proxy": false,
      "hosting": true
    },
    {
      "status": "success",
      "country": "Australia",
      "city": "Sydney",
      "isp": "Cloudflare, Inc.",
      "query": "1.1.1.1",
      "proxy": false,
      "hosting": true
    }
  ]
}
Tip: Batch lookups are more efficient when enriching multiple IPs from a single event. They use a separate rate limit (15 req/min) but each request can contain up to 100 queries.
