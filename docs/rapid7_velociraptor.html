Rapid7 Velociraptor Plugin

Manage endpoint collections, hunts, and artifact deployments using the Rapid7 Velociraptor API.

This InsightConnect plugin provides integration with Rapid7 Velociraptor, enabling security teams to automate endpoint forensics, threat hunting, and incident response workflows directly from their SOAR platform.

Key Features

  • Search and retrieve endpoint client information
  • Launch targeted artifact collections on specific endpoints
  • Create and monitor fleet-wide threat hunts
  • Retrieve collection and hunt results for investigation
  • Manage client labels for incident tracking
  • Browse available Velociraptor artifacts

Requirements

  • Rapid7 Velociraptor API key
  • Organization ID from the Velociraptor platform
  • InsightConnect orchestrator with network access to the Rapid7 platform API

Setup

Vendor Side Configuration

To configure this plugin, you need credentials from the Rapid7 Insight Platform:

  1. API Key: Navigate to the Insight Platform settings at Administration > API Keys. Generate a new Platform API key or use an existing one. This key authenticates all Velociraptor API requests made by the plugin.
  2. Organization ID: Locate your Organization ID from the Velociraptor organization settings. This identifies which Velociraptor deployment the plugin will communicate with.
Tip: Use a dedicated service account API key rather than a personal key to avoid disruption when team members rotate credentials.

InsightConnect Side Configuration

When configuring the connection in InsightConnect, provide the following fields:

FieldTypeRequiredDescription
RegionString (enum)RequiredRapid7 platform region. Options: us, us2, us3, eu, au, ca, ap, jp
Org IDStringRequiredVelociraptor organization ID (e.g., 686a33ec-88b9-4f84-bddf-2a1fb0fecb85)
API KeyCredential (Secret Key)RequiredRapid7 Platform API key for authentication

Network Requirements

Network Access: The InsightConnect orchestrator must be able to reach https://{region}.api.insight.rapid7.com on port 443 (HTTPS). Replace {region} with your configured region code (e.g., us, eu, ap).

Permissions

Rapid7 Velociraptor uses role-based access control (RBAC) to manage what actions users and API keys can perform. The three primary roles are:

  • Reader: View-only access to clients, hunts, flows, and artifacts. Cannot create or modify resources.
  • Investigator: Can create flows (collections) and hunts, manage labels, and view all resources. Suitable for active incident responders.
  • Administrator: Full access to all Velociraptor functionality including configuration and user management.

The table below maps each plugin action to the minimum required permission level:

Get ClientReaderView client details
List ClientsReaderSearch/filter clients
Get Client LabelsReaderView assigned labels
Add LabelInvestigatorModify client labels
Remove LabelInvestigatorModify client labels
Create FlowInvestigatorLaunch artifact collections
Get FlowReaderView flow status

Get Flow ResultsReaderView collection results

Recommendation: For most automation workflows, use an API key associated with the Investigator role. This provides the ability to create collections and hunts without granting full administrative privileges.

Get Client

Retrieve details for a specific Velociraptor client endpoint.

Input

NameTypeRequiredDescription
client_idstringRequiredThe unique identifier of the client

Output

Returns a client object containing the endpoint details.

{
  "client": {
    "client_id": "C.1234567890abcdef",
    "hostname": "WORKSTATION-01",
    "release": "10.0.19045",
    "architecture": "amd64",
    "client_version": "0.7.1",
    "system": "windows",
    "first_seen_at": 1700000000
  }
}

List Clients

Search and list Velociraptor client endpoints with optional filters.

Input

NameTypeRequiredDescription
hostnamestringOptionalFilter by hostname substring
osstringOptionalFilter by operating system
labelstringOptionalFilter by client label
statusstringOptionalFilter by client online status. Options: ALL, ONLINE, OFFLINE. Default: ALL
limitintegerOptionalMaximum number of results to return. Default: 100

Output

Returns a list of client objects matching the search criteria and a count of results.

{
  "clients": [
    {
      "client_id": "C.1234567890abcdef",
      "hostname": "WORKSTATION-01",
      "system": "windows",
      "first_seen_at": 1700000000
    }
  ],
  "count": 1
}

Create Hunt

Create a new fleet-wide threat hunt to collect artifacts across multiple endpoints.

Input

NameTypeRequiredDescription
hunt_descriptionstringRequiredA description of the hunt purpose
artifacts[]stringRequiredList of artifact names to collect in this hunt
os_conditionstringOptionalTarget operating system. Options: ALL, WINDOWS, LINUX, OSX. Default: ALL
include_labels[]stringOptionalOnly target clients with these labels
exclude_labels[]stringOptionalExclude clients with these labels
client_limitintegerOptionalMaximum number of clients to schedule the hunt on
expiresintegerOptionalHunt expiration time as a Unix timestamp

Output

Returns the unique identifier of the created hunt.

{
  "hunt_id": "H.abcdef1234"
}

List Artifacts

Browse available Velociraptor artifact definitions for use in collections and hunts.

Input

NameTypeRequiredDescription
artifact_typestringOptionalFilter by artifact type. Options: ALL, CLIENT, CLIENT_EVENT, SERVER, SERVER_EVENT, NOTEBOOK. Default: ALL
osstringOptionalFilter by target operating system. Options: ALL, LINUX, WINDOWS, DARWIN. Default: ALL
include_built_inbooleanOptionalInclude built-in artifacts in results. Default: true
include_custombooleanOptionalInclude custom artifacts in results. Default: true

Output

Returns a list of artifact_listing objects and a count.

{
  "artifacts": [
    {
      "name": "Windows.System.ProcessList",
      "description": "List running processes",
      "type": "CLIENT",
      "built_in": true
    },

      "name": "Custom.Lateral.Movement",
      "description": "Detect lateral movement indicators",
      "type": "CLIENT",
      "built_in": false
