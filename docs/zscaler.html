Zscaler Plugin

Rapid7 InsightConnect · v2.0.0 · Zscaler OneAPI 2024 · OAuth 2.0 Private Key JWT

Description

Zscaler is a SaaS security platform that provides fast, secure connections between client applications, regardless of device, location, or network. This plugin uses the Zscaler OneAPI unified gateway (api.zsapi.net) with OAuth 2.0 Client Credentials authentication to manage security policies across three Zscaler product lines:

  • ZIA (Zscaler Internet Access) — URL categorization, blacklisting, DLP, firewall rules, logs, users, threat feeds
  • ZPA (Zscaler Private Access) — Application segments and server groups
  • ZCC (Zscaler Client Connector) — VPN gateway bypass auditing and management

Key Features

  • Lookup categories for a given URL
  • Blacklist or unblacklist URLs
  • Create, delete, and search users
  • Get and update URL categories
  • Manage DLP incidents with time-range filtering
  • List, get, create, and update firewall rules
  • Retrieve web and firewall logs
  • Submit threat feed IoCs (IPs, domains, URLs)
  • Manage ZPA application segments and server groups
  • Audit and manage ZCC VPN gateway bypass entries

Requirements

  • A Zscaler OneAPI OAuth 2.0 Client ID from ZIdentity
  • A PEM RSA private key for JWT signing (RS256)
  • The customer vanity domain (e.g., mycompany)
  • Network connectivity from InsightConnect orchestrator to *.zsapi.net on HTTPS (port 443)

Setup & Configuration

Step 1: Create an API Client in ZIdentity

  1. Log in to ZIdentity at https://{vanity_domain}.zsapi.net
  2. Navigate to Administration > API Clients
  3. Click Add API Client
  4. Configure:
    • Name: Descriptive name (e.g., InsightConnect-SOAR)
    • Description: Purpose of the integration
    • Scopes: Assign the minimum scopes needed (see Permissions section below)
  5. Save and note the Client ID

Step 2: Generate and Upload an RSA Key Pair

  1. Generate an RSA private key (2048-bit minimum, 4096-bit recommended): 
    openssl genrsa -out zscaler_private.pem 4096
  2. Extract the public key: 
    openssl rsa -in zscaler_private.pem -pubout -out zscaler_public.pem
  3. In ZIdentity, navigate to the API Client you created
  4. Click Add Public Key and upload zscaler_public.pem
  5. Keep zscaler_private.pem secure — this is what you'll paste into the InsightConnect connection
Security: The private key should never be shared or stored in plaintext outside of InsightConnect's credential vault. Treat it like a password.

Step 3: Configure the InsightConnect Connection

FieldTypeRequiredDescription
Client IDstringRequiredOAuth Client ID from the ZIdentity API Client
Private Keycredential_asymmetric_keyRequiredPEM-encoded RSA private key for JWT signing. Paste the full content of zscaler_private.pem
Vanity DomainstringRequiredYour Zscaler vanity domain (e.g., mycompany). This is the prefix in mycompany.zsapi.net
CloudstringOptionalZscaler cloud domain. Default: zsapi.net. Most customers use the default.
How authentication works: The plugin builds a JWT assertion (signed with your private key), sends it to https://{vanity_domain}.{cloud}/oauth2/v1/token to obtain an access token, then uses that token as a Bearer header for all API requests through api.zsapi.net. Tokens are cached and auto-refreshed before expiry.

Network Requirements

  • Outbound HTTPS to {vanity_domain}.zsapi.net (token endpoint)
  • Outbound HTTPS to api.zsapi.net (unified API gateway for all ZIA/ZPA/ZCC requests)
  • No inbound connections required
  • If using an HTTP proxy, ensure it allows connections to *.zsapi.net

Get VPN Gateway Bypasses

ZCC Retrieve VPN gateway bypass entries from ZCC application profiles. Used to audit for shared-hosting/CDN IP conflicts where a bypass hostname resolves to a shared IP (e.g., Cloudflare), effectively tunneling unintended traffic outside the VPN.

Input

No input required — retrieves all profiles.

Output

{
  "profiles": [
    {
      "profile_id": "12345",
      "profile_name": "Default Profile",
      "vpn_gateways": [
        {"hostname": "gateway.example.com", "ip": "10.0.0.1", "type": "hostname"}
      ]
    }
  ]
}

Remove VPN Gateway Bypass

ZCC Remove a VPN gateway bypass entry from a ZCC application profile. Idempotent — removing an entry that doesn't exist returns success.

Input
