Halo ITSM Plugin

Halo ITSM is a cloud-based IT Service Management platform that provides ticket management, asset tracking, and service desk capabilities. This plugin enables Rapid7 InsightConnect workflows to interact with Halo ITSM tickets programmatically.

Key Features

  • Create, update, delete, and retrieve tickets
  • List tickets with filtering support (by status, type, agent, client, or search term)
  • Add actions (notes/comments) to tickets
  • Attach files to tickets
  • Manage ticket assignments and status changes

Requirements

  • Halo ITSM API Client ID and Client Secret
  • Base URL of your Halo ITSM instance
  • API application configured with the Client Credentials grant type in Halo ITSM
To configure API access, go to Configuration > Integrations > HaloITSM API > View Applications in your Halo instance and create a new application with the Client ID and Secret (Services) authentication type.

Create Ticket

Create a new ticket in Halo ITSM.

Input

NameTypeRequiredDescription
summarystringRequiredSummary of the ticket
tickettype_idintegerRequiredID of the ticket type (e.g. 1 for Incident, 2 for Service Request)
detailsstringOptionalFull details of the ticket in plain text or HTML
user_idintegerOptionalID of the user raising the ticket
client_idintegerOptionalID of the client associated with the ticket
priority_idintegerOptionalID of the ticket priority
status_idintegerOptionalID of the initial status for the ticket
category_1stringOptionalPrimary category of the ticket
category_2stringOptionalSubcategory of the ticket
agent_idintegerOptionalID of the agent to assign the ticket to
impactstringOptionalImpact level of the ticket
urgencystringOptionalUrgency level of the ticket
site_idintegerOptionalID of the site to associate with the ticket

Output

Returns the created ticket object.

{
  "ticket": {
    "id": 101,
    "summary": "Server is unresponsive",
    "details": "The production server has been unresponsive since 10:00 AM",
    "status_id": 1,
    "priority_id": 3,
    "tickettype_id": 1,
    "category_1": "Hardware",
    "agent_id": 15,
    "agent_name": "John Smith",
    "user_name": "Jane Doe",
    "client_name": "ACME Corp",
    "datecreated": "2025-07-10T10:05:00Z"
  }
}

List Tickets

List tickets from Halo ITSM with optional filtering.

Input

NameTypeRequiredDescription
status_idintegerOptionalFilter by status ID
tickettype_idintegerOptionalFilter by ticket type ID
agent_idintegerOptionalFilter by assigned agent ID
client_idintegerOptionalFilter by client ID
countintegerOptionalMaximum number of tickets to return (default: 50)
searchstringOptionalSearch string to filter tickets

Output

Returns an array of ticket objects.

      "summary": "Email not working",

      "summary": "Printer jam on 3rd floor",
      "status_id": 1,

      "client_name": "ACME Corp"

}

Add Action to Ticket

Add an action (note, comment, or update) to an existing ticket. Actions are part of Halo's audit trail architecture.

Input

NameTypeRequiredDescription
ticket_idintegerRequiredID of the ticket to add the action to
notestringRequiredContent of the action note in plain text or HTML
outcomestringOptionalOutcome of the action
hiddenfromuserbooleanOptionalWhether this note should be hidden from the end user (default: false)

Output

{
  "action": {
    "id": 501,
    "ticket_id": 1234,
    "who": "John Smith",
    "note": "Investigated the issue and found root cause",
    "hiddenfromuser": false,
    "outcome": "Ongoing",
    "actiondate": "2025-07-10T10:30:00Z"
  }
}

Attach File to Ticket

Attach a file to an existing ticket via an action note. The file is uploaded as multipart form data and linked to the ticket's action history.

Input

NameTypeRequiredDescription
ticket_idintegerRequiredID of the ticket to attach the file to
filenamestringRequiredName of the file (e.g. screenshot.png)
contentbytesRequiredBase64-encoded content of the file
notestringOptionalNote to include with the attachment
hiddenfromuserbooleanOptionalWhether the attachment action should be hidden from the end user (default: false)

Output

{
  "action": {
    "id": 502,
    "ticket_id": 1234,
    "who": "John Smith",
    "note": "Attaching error log for reference",
    "hiddenfromuser": false,
    "outcome": null,
    "actiondate": "2025-07-10T11:00:00Z"
  }
}
