Microsoft Teams Plugin

Rapid7 InsightConnect Plugin · Version 8.0.0 · Community Support

Description

Microsoft Teams is a unified communications platform. The Microsoft Teams plugin allows you to send and trigger workflows on new messages. The plugin also allows for teams management with the ability to add and remove teams, channels, and users.

This plugin uses the Microsoft Teams API (Microsoft Graph v1.0) to interact with Microsoft Teams.

This plugin uses app-only authentication (client_credentials OAuth2 flow) with Bot Framework integration. No user account is required.

Key Features

  • Communication Management for all Microsoft products
  • App-only authentication (no user account required)
  • Bot Framework integration for sending messages
  • Send plain text and HTML messages to channels and chats
  • Team and channel management (create, delete, add/remove members)
  • Trigger workflows based on new messages with regex filtering
  • Automatic extraction of security indicators (IPs, URLs, hashes, CVEs) from messages
  • Support for Commercial, GCC, GCC High, and DoD environments

Requirements

  • Azure App Registration with Application ID and Secret
  • Microsoft Graph application permissions with admin consent
  • Azure Bot registration (for sending messages)
  • An Azure account with Global Administrator or Application Administrator role
  • A Microsoft Teams environment where you can install apps

Setup & Configuration

Part 1: Create the Azure App Registration

Step 1: Register a New Application

  1. Go to the Azure Portal
  2. Navigate to Microsoft Entra ID (formerly Azure Active Directory) → App registrations
  3. Click + New registration
  4. Fill in the details:
    • Name: InsightConnect Teams Bot (or any descriptive name)
    • Supported account types: Select Accounts in this organizational directory only (Single tenant)
    • Redirect URI: Leave blank (not needed for this flow)
  5. Click Register

Step 2: Record Your Application (Client) ID

After registration, you'll land on the app's Overview page. Record these values:

FieldWhere to Find ItPlugin Connection Field
Application (client) IDOverview page, top sectionApplication ID
Directory (tenant) IDOverview page, top sectionDirectory ID

Step 3: Create a Client Secret

  1. In your app registration, go to Certificates & secrets → Client secrets
  2. Click + New client secret
  3. Set a description (e.g., InsightConnect Plugin) and expiration (recommended: 24 months)
  4. Click Add
  5. Immediately copy the Value (not the Secret ID) — this is shown only once
Important: The secret value is only displayed once. If you navigate away without copying it, you'll need to create a new one.

Part 2: Configure API Permissions

Step 4: Add Microsoft Graph Application Permissions

  1. In your app registration, go to API permissions
  2. Click + Add a permission → Microsoft Graph → Application permissions
  3. Add the permissions based on which actions you plan to use (see Permissions section below)

Step 5: Grant Admin Consent

  1. On the API permissions page, click Grant admin consent for [Your Organization]
  2. Click Yes to confirm
  3. Verify all permissions show a green checkmark under the Status column
This step requires Global Administrator or Privileged Role Administrator. If you don't have this role, you'll need to request consent from someone who does.

Part 3: Create the Azure Bot Registration

The Bot Framework is used to send messages to Teams channels and chats without a user account. Messages will appear as coming from the bot identity.

Step 6: Create a Bot Resource

  1. In the Azure Portal, search for Azure Bot in the top search bar
  2. Click Create
  3. Fill in the details:
    • Bot handle: InsightConnectBot (must be globally unique)
    • Subscription: Select your Azure subscription
    • Resource group: Select an existing one or create new
    • Pricing tier: Select Standard
    • Type of App: Single Tenant
    • Creation type: Use existing app registration
    • App ID: Paste the Application (client) ID from Step 2
    • App tenant ID: Paste the Directory (tenant) ID from Step 2
  4. Click Review + create → Create

Step 7: Configure the Bot Channel (Teams)

  1. Once the bot resource is created, go to it in the Azure Portal
  2. Navigate to Channels in the left menu
  3. Click Microsoft Teams (the Teams icon)
  4. Accept the Terms of Service
  5. Under Messaging, ensure it's enabled
  6. Click Apply

Part 4: Install the Bot in Teams

For the bot to send messages to a specific team/channel, it must be installed in that team.

Option A: Sideload a Teams App Manifest

Create a manifest.json file:

{
  "$schema": "https://developer.microsoft.com/en-us/json-schemas/teams/v1.17/MicrosoftTeams.schema.json",
  "manifestVersion": "1.17",
  "version": "1.0.0",
  "id": "YOUR_APPLICATION_CLIENT_ID",
  "developer": {
    "name": "Your Organization",
    "websiteUrl": "https://www.rapid7.com",
    "privacyUrl": "https://www.rapid7.com/privacy-policy/",
    "termsOfUseUrl": "https://www.rapid7.com/legal/"
  },
  "name": { "short": "InsightConnect Bot", "full": "InsightConnect Automation Bot" },
  "description": {
    "short": "InsightConnect SOAR automation bot",
    "full": "Enables InsightConnect to send automated messages to Microsoft Teams."
  },
  "icons": { "outline": "outline.png", "color": "color.png" },
  "accentColor": "#FF6600",
  "bots": [{
    "botId": "YOUR_APPLICATION_CLIENT_ID",
    "scopes": ["team", "personal", "groupChat"],
    "supportsFiles": false,
    "isNotificationOnly": true
  }],
  "permissions": ["identity", "messageTeamMembers"],
  "validDomains": []
}

Replace YOUR_APPLICATION_CLIENT_ID with your actual Application (client) ID.

  1. Create two icon files (32x32 outline.png and 192x192 color.png)
  2. Put manifest.json, outline.png, and color.png into a ZIP file
  3. In Microsoft Teams: go to Apps → Manage your apps → Upload an app
  4. Upload the ZIP file
  5. Add the bot to each team: go to the team → Manage team → Apps → find and add the bot

Option B: Install via Teams Admin Center

  1. Go to the Teams Admin Center
  2. Navigate to Teams apps → Manage apps
  3. Click + Upload new app → upload the ZIP package
  4. Use Setup policies to push the app to users/teams

Finding Your App Catalog ID

  1. Go to the Teams Admin Center
  2. Navigate to Teams apps → Manage apps
  3. Search for your bot app by name
  4. The App ID shown is your Teams App Catalog ID
The App Catalog ID is not the same as the Azure App Registration client ID. It's the ID assigned when the app is published to your organization's Teams app catalog.

Part 6: Verify the Setup

  1. Save the plugin connection in InsightConnect
  2. The connection test will authenticate using client_credentials flow and call the Microsoft Graph API
  3. If successful, you'll see a green "Connected" status
Testing message send: Use the Send Message by GUID action with a team and channel GUID. Right-click a channel in Teams → "Get link to channel" to extract the IDs from the URL.

Least-Privilege Guidance

Recommendation: Start with the "Notifications Only" profile and add permissions as needed. Use separate App Registrations for different privilege levels when possible (e.g., one for read-only monitoring, another for management actions).
  • TeamsAppInstallation.ReadWriteForChat.All is only required if you configure the App Catalog ID in the connection for auto-install behavior
  • TeamMember.ReadWriteNonOwnerRole.All is a least-privileged alternative to TeamMember.ReadWrite.All if you only need to add members (not owners)
  • Adding guest users to a team via application permissions is not supported by Microsoft — guest users can only be added using delegated permissions

Send Message

Send a message to a Teams channel or chat. Specify either team_name + channel_name for channel messages, or chat_id for chat messages.

Input

NameTypeRequiredDescription
team_namestringOptionalTeam name
channel_namestringOptionalChannel name
thread_idstringOptionalTo post in a thread, select parent message ID
chat_idstringOptionalThe ID of the chat
messagestringRequiredMessage to send

Output

Returns the message object that was created, including body, sender, timestamp, and web URL.

{
  "body": {"contentType": "text", "content": "Hello!"},
  "from": {"user": {"displayName": "Example User", "id": "3395856c-..."}},
  "ID": "1234567891",
  "messageType": "message",
  "importance": "normal"
}

New Message Received (Trigger)

Poll a channel for new messages and trigger a workflow when a match is found. Optionally filter messages with a regex pattern.

Input

NameTypeRequiredDescription
team_namestringRequiredTeam name
channel_namestringRequiredChannel name
message_contentstringOptionalRegex to match new messages against

Output

NameTypeDescription
messagemessageThe message object that was created (includes body, sender, timestamp, first_word, words)
indicatorsindicatorsExtracted security indicators: domains, URLs, emails, hashes (MD5/SHA1/SHA256), IPs (v4/v6), MAC addresses, CVEs, UUIDs
channel_namestringName of the channel where the message was posted
team_namestringName of the team to which the channel belongs

Example Output

{
  "message": {
    "body": {"contentType": "text", "content": "!investigate 192.168.1.100"},
    "from": {"user": {"displayName": "Example User"}},
    "first_word": "!investigate",
    "words": ["!investigate", "192.168.1.100"],
    "ID": "1234567891"
  },
  "indicators": {
    "IP Addressses": {"IPv4 Addressses": ["192.168.1.100"]},
    "Domains": [],
    "URLs": [],
    "Hashes": {"MD5 Hashes": [], "SHA1 Hashes": [], "SHA256 Hashes": []},
    "CVEs": [],
    "UUIDs": []
  },
  "channel_name": "Bot Commands",
  "team_name": "Security Operations"
}
Regex filtering: The message_content field uses Python regex syntax. Leave empty to trigger on all messages.

Troubleshooting

  • "Bot is not authorized to send messages" — Install the bot app in the target team. For chats, configure the App Catalog ID for auto-install.
  • "Authentication to Microsoft Graph failed" — Verify Application ID, Directory ID, and Secret. Ensure admin consent is granted.
  • "Forbidden - insufficient permissions" — Add the missing permission and re-grant admin consent.
  • "Resource not found" — Verify team/channel IDs and that the bot is installed in the target team.
  • "Rate limit exceeded" — Microsoft Graph limits ~10,000 requests per 10 minutes. Add delays or reduce polling frequency.
  • Duplicate team names — If multiple teams share the same name, the oldest team will be used.
